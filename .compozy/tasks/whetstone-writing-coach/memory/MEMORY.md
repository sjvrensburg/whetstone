## Shared Decisions

- Stack: TS strict, `module: commonjs`/`moduleResolution: node`; esbuild bundles `src/extension.ts`→`dist/extension.js` (vscode external) and `src/dev/main.ts`→`dist/dev/cli.js`. ESLint v8 (`.eslintrc.json`, with a `*.js` override) + Prettier.
- Module boundary is enforced: domain logic lives in `src/<module>/`; `src/extension.ts` is wiring only (DI seam `createContainer` + no-op commands + `registerViews`). Feature tasks append to `package.json#contributes` and swap a command's no-op handler for a real one wired from the container.
- Extension id `whetstone.whetstone`; engine `^1.90.0`.
- **Domain types & coaching schema are single-sourced (task 02).** Move taxonomy + length caps live only in `src/shared/constants.ts` (`OBSERVATION_KINDS`, `REFLECTION_MAX_LENGTH=280`, `QUESTION_MAX_LENGTH=200`, `MAX_OBSERVATIONS=7`); the coaching schema and validator both derive from them — never redefine. `Observation.kind` is `ObservationKind` (derived from the const array). The forced-output schema `COACHING_JSON_SCHEMA` (`src/coaching/schema.ts`) is the structural floor: `additionalProperties:false` everywhere + exactly `{anchor,kind,reflection,question}` per observation = no prose field (the F2 invariant at the type level). `validateStructuredCoaching`/`isStructuredCoaching` enforce structure + caps + interrogative-question (caps are enforced by the validator because Anthropic strips `maxLength`/`minimum` server-side).
- Coverage gate (`vitest.config.ts#coverage.include`) is an allow-list: add new logic files; do **not** add type-only files (e.g. `src/shared/types.ts` — compiles to nothing).

## Shared Learnings

- Two harnesses, kept apart: vitest runs `test/unit/**` (never imports `vscode`); integration runs `src/test/**` via `@vscode/test-electron` (Mocha), compiled separately by `tsconfig.test.json`→`out/`, launched by `out/test/runTest.js`.
- Unit-testing a `vscode`-coupled module: vitest aliases `vscode`→`test/support/vscode-stub.ts`. Extend that stub when a unit test needs a new API member.
- Coverage gate (≥80%) uses an explicit allow-list in `vitest.config.ts` (`coverage.include`). **Add each new logic file there or it is silently excluded from the gate.**
- Offline provider tests use `test/support/fixtures.ts`: `FixtureRecorder.record(method,req,liveFn)`→`save(file)`; `FixtureReplayer.fromFile(file)` + `createReplayCall<Req,Res>(method)` is the deterministic drop-in stub for `coach`/`judge` (tasks 09/11/19).
- `ProviderResult<T>` (`{ok:true,value}|{ok:false,error}`) is the universal return type for provider calls — callers must always unwrap, never treat as bare value.
- Pre-existing tsc errors in `test/unit/guard-deterministic.test.ts` (unused var `_shortDoc`, type mismatch on line 158) — from task 10, not introduced by task 11.

## Handoffs

- Task 07 (`src/ledger/*`) consumes the task-03 primitives from `../shared`: `chainHash(entry)` takes the entry **without** its `hash` field (it already carries `prevHash`) and returns `SHA-256(prevHash + canonicalize(entry))` — call it once per appended entry, set `prevHash:""` for genesis. Checkpoints: `sign(latestHash, privateKeyPem)`→base64 sig; `verify(latestHash, sig, publicKeyPem)`→bool that returns `false` (never throws) on bad/wrong/malformed input. Keys are PEM (SPKI public / PKCS#8 private) ready for SecretStorage. `canonicalize` is the only stringifier the chain may use — never `JSON.stringify` (key order would break `verify()`).
