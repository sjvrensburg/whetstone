Friction, not proof

Four ways it slows you down on purpose

Each instrument responds to a friction dial from Quiet to Deep Work, so you choose how much resistance the draft puts up.

paste, held for review

Paste quarantine

Anything you paste is held apart and marked. To release it, rewrite it in your own words or tag it as a quotation. Borrowed text cannot quietly become your draft.

claim to own

Claim to own

Ownership is measured as how much of the original paste survives your rewriting, not the reverse. State your claim, and it travels with the draft into the disclosure.

teach-back checkpoint

Teach-back

At a cadence you set, Whetstone pauses and asks you to summarize the argument so far in your own words before you keep writing.

coach: question-only

A coach that only asks

The optional coach reads your draft and replies with questions. A deterministic guard screens every reply, so it can sharpen your thinking but never hand you sentences.

The writing surface

A plain editor, a live preview

No modes to memorize and nothing to learn from a manual. It is mouse and keyboard driven in the Fresh and Micro tradition: menus, theming, click to place the cursor, drag to select.

Live Markdown and Quarto preview. Headings, emphasis, code, blockquotes, and LaTeX math rendered as you type.
Local grammar via Harper. Issues are underlined and collected in a Suggestions pane. Apply a fix with Enter. No text leaves your machine.
Outline, find, replace, go to line. Jump by heading, search the draft, and render with Quarto from a single key.
suggestions · harper

The coach

It can only ask questions

The coach is optional and off until you point it at an endpoint. When it is on, it is question-only by construction, not by polite request.

A guard that always runs. Every reply is screened for length, rewrite and dictation patterns, n-gram overlap with your draft, and forbidden labels before you ever see it.
An optional second opinion. A response judge (a second model) can re-screen each reply and only ever withhold it. If it cannot be reached, the coach fails open on the guard that already passed.
Your endpoint, your keys. Point it at a local server or a hosted provider. An API key field accepts an env:NAME reference, so a secret never has to touch disk.
ProtocolsOpenAI-compatible Chat Completions, Anthropic Messages
ScreeningDeterministic guard (always), optional LLM judge (defence in depth)
HistoryPer document, owner-only on disk. The draft itself is never stored here.
ai settings · ctrl+e
Screen a reply without opening the editor
# is this reply allowed against the draft? prints a JSON verdict
whetstone-tui guard --reply "Have you considered the counterargument in section 2?" \
  --draft essay.qmd

# one guarded (and optionally judged) coaching turn, headless
whetstone-tui coach essay.qmd --message "Where is my argument weakest?"

An honest record

Export how it was written

An append-only journal records the shape of your process: paste sizes, claims, attributions, and edit bursts. It holds metadata only. Your prose is never written to it. From that journal you can render a plain disclosure document, a self-reported account, not a verdict on who you are.

process journal · ctrl+p

The journal

A timeline of what happened, kept to sizes, locations, and your own stated claim. Metadata only, by design.

disclosure preview · ctrl+d

The disclosure

A readable account you can hand in alongside the work. It says how the draft came together. It never claims to verify authorship or personhood.

Themes

Six built-in themes, switched live

Press Ctrl+T to preview and switch. The choice persists between sessions.

Scriptable

Drive the same core from a script

Every guarantee in the editor is also a subcommand that prints JSON, so an agent, a script, or a CI step can run the same checks. A bare file path opens the editor.

whetstone-tui --help
whetstone-tui lint essay.qmd                              # Harper diagnostics + fixes
whetstone-tui coach essay.qmd --message "…"             # one guarded coaching turn
whetstone-tui guard --reply "…" --draft essay.qmd        # screen an arbitrary reply
whetstone-tui ownership --original a.txt --current b.txt  # claim-to-own survival
whetstone-tui disclosure --journal events.json --doc-id essay.qmd

Bring back the friction

Whetstone is a single Rust binary. It needs a recent toolchain (edition 2024). Quarto is optional and only used when you render with Ctrl+R.
